# AI-ассистент для дистрибьютора косметики

CLI-ассистент, который собирает данные о клиентах, товарах и заказах из разных источников, приводит их к единому виду и отвечает на бизнес-вопросы через LLM + function calling.

Цель - не заменить менеджера, а сократить ручную сверку между таблицами, остатками и историей заказов.

## Что умеет

### 1. Кому написать и что предложить

Ассистент собирает профиль конкретного клиента: заметки менеджера, историю completed-заказов, последний заказ, любимые бренды/категории, средний чек и проблемы данных по этому клиенту.

Почему выбрал: менеджеру Юле важно быстро подготовиться к контакту с клиентом, не открывая отдельно клиентов, заказы и товары. В `clients.notes` уже есть полезные сигналы вроде регулярности покупок, крупных чеков и сезонных поводов.

Пример:

```bash
python3 main.py "Что знаем про Lana и что ей предложить?"
```

### 2. Какие товары рискуют закончиться

Ассистент сравнивает текущий остаток товара с фактическими продажами за период и показывает товары, где маленький остаток действительно важен из-за спроса.

Почему выбрал: смотреть только на `stock_quantity` недостаточно. Остаток 10 может быть безопасным для редкого товара и критичным для ходовой позиции. Поэтому сценарий связывает остатки из `products` с продажами из `orders`.

Пример:

```bash
python3 main.py "Какие товары могут скоро закончиться?"
```

### 3. Что произошло в продажах за период

Ассистент делает короткую управленческую сводку: выручка, количество заказов, топ клиенты, топ товары, статусы заказов, складские риски и проблемные строки в данных.

Почему выбрал: Ирине как владельцу нужен быстрый обзор бизнеса без ручных подсчётов. При этом важно видеть не только сумму продаж, но и доверие к данным: некорректные даты, отрицательные суммы, неизвестных клиентов.

Пример:

```bash
python3 main.py "Посчитай выручку за весь период"
```

Курьер не выбран как целевой пользователь MVP, потому что в предоставленных данных нет маршрутов, адресов доставки, времени доставки или статусов отгрузки. Поэтому сценарии сфокусированы на Юле и Ирине, для которых есть данные по клиентам, заказам и остаткам.

## Как запустить

1. Установить зависимости:

```bash
pip install -r requirements.txt
```

2. Создать `.env`:

```bash
cp .env.example .env
```

3. Заполнить ключи:

```text
OPENAI_API_KEY=...
SUPABASE_URL=...
SUPABASE_ANON_KEY=...
ORDERS_CSV_URL=...
```

`SUPABASE_URL`, `SUPABASE_ANON_KEY` и `ORDERS_CSV_URL` опциональны для локальной проверки: если внешние источники недоступны, приложение использует fallback-файлы из `test-data/`.

4. Запустить CLI:

```bash
python3 main.py
```

Можно передать вопрос сразу:

```bash
python3 main.py "Дай сводку продаж за 30 дней"
```

Для выхода из интерактивного режима: `exit`, `quit` или `выход`.
