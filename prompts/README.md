# Промпты

Складывай промпты сюда — отдельными файлами на каждый сценарий или модуль.

## Принципы (рекомендация)

1. **Структура:** роль → контекст → инструкция → формат ответа → примеры
2. **Версионируй** в git — каждое изменение промпта это отдельный коммит с описанием, что меняли и зачем
3. **Не вшивай данные в промпт.** Используй плейсхолдеры `{client_data}`, `{products}` и подставляй из кода
4. **Не доверяй LLM считать.** Числа считай в коде, LLM — за формулировку и интерпретацию

## Пример организации

```
prompts/
  scenario_1_top_clients.md     # промпт для сценария «топ клиентов»
  scenario_2_low_stock.md       # промпт для сценария «низкие остатки»
  scenario_3_birthday.md        # промпт для сценария «ДР клиента»
  system_base.md                # общий системный промпт (если есть)
```

Удали этот файл и переделай папку под себя.
